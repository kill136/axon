# Axon Commercial License

## Why a commercial license?

Axon's source code is available under the [GNU Affero General Public License v3.0 (AGPL-3.0)](LICENSE). The AGPL requires that if you modify Axon or use it to provide a network service, you must release your complete source code under the same license.

If your use case cannot comply with the AGPL — for example, you want to keep your code proprietary, embed Axon in a closed-source product, or offer it as a managed SaaS — you need a commercial license.

## When do you need a commercial license?

| Use case | AGPL (free) | Commercial license needed |
|---|---|---|
| Personal use, learning, experimentation | Yes | No |
| Academic research | Yes | No |
| Open-source project (AGPL-compatible) | Yes | No |
| Internal company tool (no modifications, no network service) | Yes | No |
| Modified version kept proprietary | No | **Yes** |
| SaaS / hosted service built on Axon | No | **Yes** |
| Embedded in a closed-source product | No | **Yes** |
| OEM / redistribution without AGPL compliance | No | **Yes** |

## What the commercial license includes

- Permission to use, modify, and distribute Axon without AGPL obligations
- Freedom to keep your source code proprietary

## FAQ

**Q: Can I try Axon for free before purchasing a commercial license?**
A: Yes. The AGPL-licensed version is fully functional. You can evaluate it freely. The commercial license is only needed when you deploy in a way that conflicts with AGPL terms.

**Q: Do contributors need a commercial license?**
A: No. Contributors to the Axon open-source project are covered by the project's contributor agreement.

**Q: What about existing MIT-licensed versions?**
A: Versions released before this license change (up to v2.2.0) remain under the MIT license. The AGPL + commercial dual licensing applies to subsequent releases only.
